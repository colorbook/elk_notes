1. 撰寫LS設定檔

設定檔區塊

分成三部分： input－ filter－ output

  • input：stdin, file, redis, sqlite, tcp
  • filter：csv, json, grok, ruby, geoip
  • output：stdout, file, elasticsearch,mongodb,nagios

設定檔範例

input(讀取CSV檔)－filter(切割欄位)－output(螢幕輸出)

input{
        #接收檔案資料
        file{
                path => "/opt/ELK/data/data_2016m1.csv"        #檔案路徑
                start_position => "beginning"                                #讀取檔案內容
                sincedb_path => "/opt/ELK/data/.sincedb*"    #sincedb記錄存放位置
        }
}

filter{
        #資料類型CSV設定
        csv{
                columns => [ "event_id","file_md5",    #欄位名稱
                "ip_src","ip_dst","username",
                "tstamp","url" ]
                separator => ","                                            #切割符號
        }
}

output{
        #以rubydebug格式輸出至螢幕
        stdout{ codec => rubydebug }
}

1. 撰寫LS設定檔

以資料時間取代系統時間

filter{
        date{
                match => ["tstamp", "YYYY-MM-dd HH:mm:ss.SSSSSSZ"]
        }
}

輸出資料至Elasticsearch

output{
        stdout{ codec => rubydebug }

        #資料輸出至ES
        elasticsearch{
                hosts => ["10.3.76.123"]                                    #指定ES IP位置預設走9200 port
                index => "elk-demo-%{+YYYY.MM.dd}"    #ES建立index名稱
        }
}

GEOIP

在Kibana地圖上呈現GeoIP位置

Logstash設定檔

在filter加入geoip

filter{
        csv{
                columns => [ "event_id","file_md5",
                        "ip_src","ip_dst","username",
                        "tstamp","url" ]
                separator => ","

                convert => { "event_id" => "integer" }
        }

        geoip{
                source => "ip_src"
                target => "geoip"
                database => "/opt/ELK/logstash/geoipDB/GeoLiteCity.dat"
        }

}

Elasticsearch Template
